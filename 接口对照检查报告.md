## 四、发现的问题汇总

### 问题 1: Logout 路径大小写不一致 (已修复)

| 项目 | 路径 |
|------|------|
| GoView | `/api/goview/sys/logout` (小写 L) |
| LED 原始 | `/api/goview/sys/Logout` (大写 L) |

**状态**: ✅ 已修复 - 修改 `led/src/routers/led/led.js` 第 12 行

---

### 问题 2: 模型字段大小写映射 (已修复)

**原始问题**: Sequelize 使用 `raw: true` 查询时返回数据库列名 (大写)，与 GoView 期望的小写字段名不匹配。

**修复方案**: 为 `Led_Projects` 和 `Led_Projectdatas` 模型添加 `field` 属性映射。

**修改文件**:
- `led/src/models/self/Led_Projects.js`
- `led/src/models/self/Led_Projectdatas.js`

**状态**: ✅ 已修复

---

### 问题 3: 上传文件缺少 fileurl 字段

**问题描述**: GoView 期望返回 `fileurl` 字段，但 LED 只返回 `fileName`。

**当前返回**:
```json
{
  "fileName": "xxx.png",
  "id": "20260331120000",
  ...
}
```

**GoView 期望**:
```json
{
  "fileName": "xxx.png",
  "fileurl": "http://..."  // 缺少此字段
}
```

**建议修复**: 在 LED 上传接口中添加 fileurl 字段，值为完整访问路径。

**状态**: ⚠️ 待修复

---

### 问题 4: 中文乱码 (字符集问题)

**问题描述**: 创建项目时中文参数出现乱码。

**测试**:
```
输入: {"projectName":"测试项目1","remarks":"测试简介"}
存储: ProjectName = "������Ŀ1", Remarks = "���Լ��"
```

**分析**: MySQL 连接字符集配置正确 (`utf8mb4`)，问题可能在于 Sequelize 或 Node.js 层面。

**状态**: ⚠️ 待进一步调查
